ASE — AI 联网搜索服务

无状态 REST API：Bearer 鉴权，POST /v1/search 返回整理后的 Markdown，供各类 AI Agent 与工具链作为检索上下文。支持自建 OpenSearch 索引优先与多引擎回落。

项目介绍 技术栈 API 端点 Agent Skill 安装与指令 调用示例 文档

项目介绍

ASE（本仓库）面向开发者与 Agent 场景：不提供传统「搜索门户」UI，而是输出可直接并入模型上下文的 Markdown 正文。服务可结合境内自建 OpenSearch做索引优先检索；索引不足时再回落到配置的搜索引擎（如 Tavily、无头浏览器抓取等，由部署环境决定）。

典型用法：将下方 Skill 安装到你所用的 AI Agent / IDE（Cursor、Claude Code、Copilot、自建 Agent 等）要求的技能目录，由 Agent 按约定调用 ASE；或通过 HTTP 客户端自行集成。

  • 同步 API，成功路径为 200 + Markdown；错误为 application/problem+json。
  • 限流保护 /v1/*；/、/api/info、/health、/metrics、/skills/* 无鉴权、不限流。
  • 开源许可：MIT（仓库根目录 LICENSE）。

技术栈（摘要）

运行时

  • Go（版本见仓库 go.mod）
  • HTTP 路由：chi v5
  • 可选：OpenSearch 2.x（opensearch-go）

可观测

  • GET /metrics — Prometheus（如 ase_search_orchestration_total、ase_index_writeback_total）
  • GET /health — 探活 JSON

HTTP API（v1）

方法路径说明
GET/本主页（HTML）
GET/api/infoJSON 服务发现（无鉴权）
POST/v1/search主业务：JSON 查询 → text/markdown（需 Bearer）
POST/v1/documents可选：写入索引字段 id / title / body_text；未配置 OpenSearch 时 501
GET/health探活，无鉴权
GET/metricsPrometheus 文本，无鉴权
GET/skills/ase-search-api/SKILL.mdAgent Skill 主文件（下载，与 API 同源）
GET/skills/ase-search-api/reference.mdSkill 参考片段
GET/skills/ase-search-api/bundle.zip上述两文件打包

完整契约与错误码见 docs/SEARCH_API_V1.md（GitHub）。

Agent Skill 配置（通用）

仓库内源文件位于 .cursor/skills/ase-search-api/（目录名中的 .cursor 仅为仓库路径，不表示仅限 Cursor 使用）。主文件为 SKILL.md，任选支持自定义技能 / 规则 / 工具说明的 Agent，将文件复制到该产品文档规定的目录即可。

Skill 元数据（YAML 头）

---
name: ase-search-api
description: >-
  Calls the ASE REST API for agent-oriented web search: POST /v1/search returns Markdown;
  optional index ingest via POST /v1/documents. Use when the user needs ASE search, asks to
  query the ASE service, or mentions ASE_BASE_URL / DEV_API_KEY / OpenSearch-backed search.
---

环境变量（调用前）

变量说明
ASE_BASE_URL服务根地址，默认 http://127.0.0.1:18080，无末尾斜杠
ASE_API_KEY / DEV_API_KEY与部署一致；请求头 Authorization: Bearer <token>

Agent 工作流（摘要）

  1. 向用户确认 ASE_BASE_URL 与密钥来源。
  2. 不确定连通性时先请求 GET …/health。
  3. POST /v1/search，body 中 query 填用户问题；providers 仅在使用者指定引擎时填写；deepsearch 可单次覆盖是否对结果 URL 再抓取正文。
  4. 将返回的 Markdown 交给用户或按需摘要；错误时解析 Problem JSON 的 title / detail。

安装路径（按你所用的 Agent 选择）

  • 通用：在 Agent 要求的「skills / rules / plugins」目录下新建 ase-search-api/，放入 SKILL.md（及可选 reference.md）。
  • Cursor 示例 — Windows：%USERPROFILE%\.cursor\skills\ase-search-api\SKILL.md；macOS/Linux：~/.cursor/skills/ase-search-api/SKILL.md。
  • 其他产品：以官方文档为准；保持 SKILL.md 内 YAML 与正文语义不变即可。

一键生成：给 AI Agent 的安装指令

填写参数后复制下方文本，粘贴到你所用产品的 Chat / Agent / 助手（如 Cursor、Claude、Copilot、自建 Agent 等），由助手按说明创建目录并写入 SKILL.md，或自行下载后放到对应技能目录。

PowerShell：仅下载 SKILL.md（可选）

调用示例

健康检查（bash）

curl -sS "${ASE_BASE_URL:-http://127.0.0.1:18080}/health"

搜索（bash）

curl -sS -X POST "${ASE_BASE_URL:-http://127.0.0.1:18080}/v1/search" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ${ASE_API_KEY}" \
  -d '{"query":"hello","providers":["stub"]}'

搜索（PowerShell）

$base = $env:ASE_BASE_URL; if (-not $base) { $base = "http://127.0.0.1:18080" }
$key = $env:ASE_API_KEY
Invoke-RestMethod -Uri "$base/v1/search" -Method Post -ContentType "application/json" `
  -Headers @{ Authorization = "Bearer $key" } -Body '{"query":"hello","providers":["stub"]}'

Docker 一次编排（仓库根目录）

docker compose up --build -d

默认暴露 18080（API）与 9200（OpenSearch）。详见 README.md。
